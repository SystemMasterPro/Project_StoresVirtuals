BIENVENIDO

Estas en el panel de administracion de una linea de tiendas, se esta simulando un sistema para una linea de tiendas virtuales, el cual permite agregar, editar, eliminar y listar la lista de tiendas, cajeros, clientes, productos, proveedores, notas de venta, etc.

Luego de realizar lo que guste el administrador en este panel de administracion, los cambios surgiran efecto en tiempo real para el cliente final, ya que este sistema genera una api, la cual esta siendo consumida con otra tecnologia, brindandole al usuario final una experiencia unica asi como eficiente de manipular.

PROYECTO FIN DE CICLO

INVERSION Tiempo nada mas

PROCEDIMIENTOS

Etapa 1
Tiempo estimado 1 dia.

  • - Realizarlo en el lenguaje de programación PYTHON con el framework DJANGO VERSION 3.0
  • - Integrarlo con la base de datos MYSQL O POSTGRES
  • - Realizar la parte del FRONTEND integrado BOOTSTRAP, PLANTILLAS etc.
  • - Generar un APIREST de todas las clases utilizando la librería REST_FRAMEWORK
  • - Instalar la librería DJANGO-SUIT para mejorar la vista del panel de administración de Django.
  • - Como mínimo van a utilizar 5 clases para desarrollar dicho software.

Etapa 2
Tiempo estimado 1 dia.

  • - Instalar y gestionar una máquina virtual (VMware WorkStation) donde van a instalar la distribución de Linux (Ubuntu 20.04 LTS)
  • - Instalar la herramienta de gestión de servidores (Putty) y acceder al servidor (Ubuntu) a través del protocolo SHH Port 22
  • - Instalar el gestor de base de datos en Ubuntu (Mysql/Postgres)
  • - Instalar el servidor Nginx
  • - Instalar el servidor Gunicorn
  • - Gestionar cada uno de los servicios y levantar el proyecto desarrollado en Python/Django

Etapa 3
Tiempo estimado 1 dia.

  • - Crear una cuenta gratuita en la plataforma HEROKU https://www.heroku.com
  • - Gestionar los servicios para alojar el proyecto desarrollado en Python/Django.
  • - Finalmente, subir el proyecto utilizando comandos básicos de git a la plataforma Heroku.

Etapa 4
Tiempo estimado 1 dia.

  • - Realizar la demostración de la funcionalidad del software a través de un video subido en la plataforma YOUTUBE de la demostración tanto para la parte local VitualBox/Ubuntu como en producción Plataforma/Heroku y cargar el código en un archivo comprimido a la plataforma Moodle.
